## What is this?

Blacksmithers is an open-source community of engineers who believe the software industry is solving the wrong problem.

The industry glues LLMs onto human-era processes and calls it innovation. We call it **Hope-Driven Architecture.**

We build the alternative: deterministic infrastructure where AI agents are native operators, not guests in a human ceremony.

This repo is our **community hub** — the place where smiths discuss architecture, share blueprints, propose new tools, and find each other.

---

## The Enemy

> **Hope-Driven Architecture** — the industry-wide habit of gluing LLMs onto human-era processes and calling it innovation.

- Wrapping `streamChatCompletions` and branding it "AI-powered observability"
- Shoving a chat assistant on 2,000 ungoverned database tables and calling it "data insights"
- 40-person governance committees producing 5 "Single Sources of Truth" for the same entity
- Giving agents more autonomy instead of more constraint
- Expanding context windows to 1M tokens instead of decomposing cognition

---

## What We Believe

```
Not better tickets   — schemas.
Not bigger context   — decomposition.
Not more autonomy    — architectural constraint.
Not faster humans    — no humans in the loop at all.
```

> *"The real question isn't 'how do we make AI work within our processes?' It's 'what processes would we design if humans weren't the bottleneck?'"*

## You Might Be a Blacksmither If...

- You cringe when you see a chat completion wrapper sold as an "AI product"
- You've said "the problem isn't the model — it's the process we're feeding it"
- You've built internal tools that outperform vendor solutions
- You believe the best agent orchestration looks nothing like human project management
- You've spent more time designing the constraint system than the agent itself
- You had the answer — the real, working, tested answer — and someone said "that's not your area." You built it anyway.

## You're NOT a Blacksmither If...

- You want AI to make you money but don't want to understand what's under the hood
- You call yourself an AI builder but can't read the code your agent writes
- You think process is bureaucracy
- You chase tools, not understanding
- You think prompt engineering is the endgame
- You've never shipped under load

No judgment. But **Blacksmithers is for people who want to understand the fire, not just warm their hands by it.**

## Code of the Smith

**I.** Constraint over autonomy — an unconstrained agent isn't powerful, it's lost.

**II.** Decomposition over context — if you need a million tokens, your architecture failed.

**III.** Schema over prose — agents deserve specs, not stories.

**IV.** Process over model — a mediocre model in a great process beats a great model in chaos.

**V.** Measure or it didn't happen — no vibes. Throughput, defect rate, cycle time. Numbers or silence.

**VI.** Backend first — control lives in the server. Everything else is decoration.

**VII.** Ship or shut up — blog posts don't deploy. Conference talks don't pass tests. The forge is hot. Hit the metal or leave.
